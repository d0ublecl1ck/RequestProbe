## Why

当前前端已经具备“字段探针 / 资源监听 / 设置”三个工作区，但整体仍然是浅色玻璃卡片风格，页面节奏、色彩层级、导航气质和交互反馈都与 `DESIGN.md` 定义的 Ferrari 式黑白编辑叙事明显不一致。用户已经补充了新的设计基线，因此需要把本项目前端从“组件拼装感”重构为“统一的桌面工作台体验”，并覆盖每个页面的可读性、层级和交互。

## What Changes

- 重构前端全局视觉系统，包括颜色 token、字体层级、按钮/卡片/输入框样式、明暗区块节奏和通用动效。
- 重构主应用壳层和侧边栏导航，使其符合黑色电影感外壳 + 白色编辑内容区的双层结构，并强化当前工作区的状态识别。
- 重新设计“字段探针”“资源监听”“设置”三个工作区的页面结构、信息层级和关键交互，确保每个页面都具备明确的标题、说明、主操作区、结果区和空状态。
- 优化字段探针右侧结果浏览方式与资源监听页面的任务控制区，使高频操作、状态反馈和长内容浏览更清晰。

## Capabilities

### New Capabilities
- `editorial-frontend-theme`: 定义前端整体采用黑白编辑式工作台视觉语言，并要求所有工作区页面遵循统一的层级、节奏和交互反馈。

### Modified Capabilities
- `workspace-navigation`: 工作区导航需要升级为更强层级的编辑式侧边栏体验，同时继续承担工作区切换职责。
- `request-lab-right-panel-tabs`: 字段探针结果区需要在保留现有分组语义的前提下强化浏览和切换反馈。

## Impact

- 影响 `frontend/src/App.jsx`、全局样式文件、侧边栏与共享容器组件，以及三个工作区页面的大部分 JSX 结构和样式。
- 不涉及 Go / Python / Wails 后端接口契约修改，但会影响前端构建产物和页面交互表现。
