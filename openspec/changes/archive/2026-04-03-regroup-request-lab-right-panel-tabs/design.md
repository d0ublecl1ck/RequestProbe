## Context

字段探针页面右侧区上一轮已经从纵向堆叠卡片改造成平级 Tabs，但现在的四个页签仍然偏细。结合当前业务流，`Python 代码` 与 `请求测试结果` 都属于“请求级验证与复用”；`测试摘要` 与 `简化代码` 都属于“字段分析后的结论产物”。因此更合理的结构是两组分栏，而不是四个并列入口。

本次变更仍然只影响 React 前端和 OpenSpec 文档，不涉及 Go、Python 或数据结构变化。

## Goals / Non-Goals

**Goals:**

- 将右侧区简化为两个顶层结果分组 Tab。
- 保持默认聚焦在第一组，符合请求解析后的首要阅读路径。
- 在字段分析完成后自动切换到第二组，直接呈现摘要与简化代码。
- 尽量不改动现有卡片内容和业务逻辑，只调整页签组织与跳转状态值。

**Non-Goals:**

- 不将每组内部再做新的子 Tabs。
- 不调整左侧输入区与字段配置区。
- 不改变单次测试、字段分析、复制代码等业务行为。

## Decisions

### 1. 顶层 Tabs 从四项收敛为两项

新的两个页签分别命名为：

- `请求代码与测试`
- `测试摘要与简化代码`

这样保留显式导航，同时把高度相关的信息放在同一阅读区。

### 2. 每个分组页签内部仍按原顺序展示两张卡片

第一组内顺序为 `Python 代码` → `请求测试结果`；第二组内顺序为 `测试摘要` → `简化代码`。这样既满足“组合一下”的要求，也保留原有内容布局和按钮位置。

### 3. 自动跳转目标改为第二组分栏

字段分析成功后，不再切到单独的 `测试摘要` 页签，而是切到 `测试摘要与简化代码` 组。这样既能立即看到摘要，也能让简化代码紧随其后可见。

## Risks / Trade-offs

- [单个分组内部仍需要滚动] → 接受该折中，因为页签数量减少是这次更重要的目标。
- [组名过长影响横向空间] → 使用两个短句式分组名称，避免四个按钮同时占宽。
- [自动跳转后摘要不在首屏] → 保持第二组中摘要卡片位于简化代码之前。

## Migration Plan

1. 更新 OpenSpec proposal、design、tasks 和 delta spec。
2. 将右侧受控 Tabs 值改成两组分栏。
3. 调整字段分析完成后的自动跳转目标。
4. 重新生成 Wails bindings 并跑前端构建验证。
