## MODIFIED Requirements

### Requirement: 用户可以创建资源监听任务
系统 MUST 提供一个资源监听入口，允许用户输入目标 URL、选择需要监听的常见文件后缀，并启动一个新的浏览器监听任务。启动任务时系统 MUST 生成唯一任务 UUID，并为该任务分配应用数据目录下的独立下载文件夹。系统 MUST 同时提供“监听所有标签页”选项，默认开启。

#### Scenario: 使用默认值启动全标签页监听
- **WHEN** 用户首次打开资源监听页面并直接使用默认配置启动任务
- **THEN** “监听所有标签页”选项 MUST 默认为已勾选
- **THEN** 当前任务 MUST 记录该监听范围配置

#### Scenario: 手动关闭全标签页监听
- **WHEN** 用户取消勾选“监听所有标签页”后启动任务
- **THEN** 系统 MUST 仅监听启动任务时绑定的当前标签页
- **THEN** 后续新增标签页 MUST 不自动进入当前任务监听范围

## ADDED Requirements

### Requirement: 多标签页监听范围必须可配置且覆盖后续新增标签页
系统 MUST 支持用户选择当前任务的监听范围。启用“监听所有标签页”时，系统 MUST 同时监听当前浏览器会话中的所有已存在标签页，并自动接管任务开始后新增的标签页；关闭该选项时，系统 MUST 保持单标签页监听行为。

#### Scenario: 已存在多个标签页时启动全标签页监听
- **WHEN** 用户在浏览器中已经存在多个标签页，并以“监听所有标签页”为已勾选状态启动任务
- **THEN** 系统 MUST 为当前浏览器会话中的每个现有标签页建立监听
- **THEN** 来自这些标签页的请求与资源 MUST 进入同一个任务列表

#### Scenario: 监听过程中出现新标签页
- **WHEN** 当前任务以“监听所有标签页”为已勾选状态运行中，用户又打开了一个新的标签页
- **THEN** 系统 MUST 自动为该新标签页建立监听
- **THEN** 该标签页后续产生的请求与资源 MUST 进入当前任务列表
