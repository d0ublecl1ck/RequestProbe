## Why

当前应用通过顶部 Tabs 在“字段探针”和“资源监听”两个工作区之间切换。随着页面内容增加，顶部导航会和内容标题区争夺视觉层级，也让工作区切换缺少一个稳定的全局导航位置。为了让桌面端结构更像工具型工作台，需要把工作区入口收拢到左侧侧边栏。

## What Changes

- 将主界面的工作区切换从顶部 Tabs 改为左侧侧边栏导航。
- 删除顶部品牌区和 GitHub 按钮，避免与工作区内容重复占据头部空间。
- 保持现有两个工作区的业务逻辑、状态管理和内容组件不变，只调整外层布局与导航形式。
- 本次变更仅面向桌面布局，不包含移动端适配。

## Capabilities

### New Capabilities

- `workspace-navigation`: 提供桌面工作区侧边栏导航能力。

### Modified Capabilities

- 无

## Impact

- 前端主入口 `App.jsx` 的外层布局会从“顶部导航 + 内容”改为“左侧导航 + 右侧内容”。
- 当前工作区状态仍由已有的 `activeWorkspaceTab` 控制，不新增路由或新的状态容器。
- 不影响 Go、Python、Wails bindings 和现有业务接口。
