## Context

当前 `frontend/src/App.jsx` 已经把两个工作区内容都组织在同一个 React 组件中，并通过 `activeWorkspaceTab` 控制顶部 Tabs 的显示。也就是说，这次需求本质上不是新增工作区，而是把导航容器从横向顶部切换改成纵向侧边栏切换。

已确认的约束：

- 删除顶部品牌区和 GitHub 按钮。
- 保留现有两个工作区：`字段探针`、`资源监听`。
- 右侧内容区尽量复用现有 JSX 结构和交互逻辑。
- 本次不处理移动端适配。

## Goals / Non-Goals

**Goals:**

- 在桌面端提供稳定的左侧工作区导航。
- 让导航区和内容区形成清晰的两栏层级。
- 尽量减少对现有业务逻辑和组件树的扰动。

**Non-Goals:**

- 不新增第三个工作区或新的页面路由。
- 不重构请求分析和资源监听内部组件。
- 不处理移动端折叠、抽屉或响应式布局。

## Decisions

### 1. 保留现有受控状态，替换导航呈现层

继续使用 `activeWorkspaceTab` 作为唯一工作区切换状态，只把顶部 `TabsList/TabsTrigger` 替换为左侧按钮式导航。这样可以避免重写状态同步逻辑，也不需要把两块内容拆成新的路由层。

### 2. 使用固定两栏工作台布局

桌面端主结构改成：

```text
+----------------------+----------------------------------+
| Sidebar Navigation   | Active Workspace Content        |
| - 字段探针           | - 请求分析内容                  |
| - 资源监听           | or                              |
|                      | - 资源监听内容                  |
+----------------------+----------------------------------+
```

左侧侧边栏宽度固定，右侧内容区自适应填满剩余空间。由于用户明确表示移动端可以不管，因此不为窄屏增加额外布局分支。

### 3. 删除头部品牌区，让工作区标题回到内容内部

顶部品牌区和 GitHub 按钮完全移除。每个工作区继续在自己的内容区顶部显示标题和说明，从而避免一个全局头部与局部标题并存。

## Risks / Trade-offs

- 由于保留了现有两个 `TabsContent` 内的大块 JSX，`App.jsx` 仍然偏长，但这是本次最稳妥的最小改动。
- 不处理移动端意味着窄窗口下布局可能拥挤，但这是本次需求明确接受的范围。

## Verification

1. 点击左侧 `字段探针` 与 `资源监听` 时，右侧内容能正确切换。
2. 页面中不再出现顶部品牌区和 GitHub 按钮。
3. 前端构建通过，说明 JSX 结构调整未破坏应用入口。
